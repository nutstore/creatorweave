# CreatorWeave 设计规范

**创建日期**: 2026-02-08
**版本**: 1.0
**状态**: 进行中

## 1. 设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **渐进式披露** | 不一次性展示所有功能，引导用户逐步发现 |
| **一致性** | 统一的视觉语言和交互模式 |
| **可访问性** | 包容所有用户，包括有特殊需求的用户 |
| **反馈明确** | 每个操作都有清晰的反馈 |

### 1.2 用户场景定位

| 用户类型 | 核心需求 | 设计重点 |
|---------|---------|---------|
| **开发者** | 代码理解、调试、重构 | 快速导航、精确定位 |
| **数据分析师** | 数据探索、可视化、报告 | 图表展示、数据清晰度 |
| **学生/研究员** | 文档阅读、学习、知识整理 | 专注模式、引用追踪 |
| **办公人员** | 文档处理、报告生成、内容创作 | 批量操作、模板支持 |

## 5. 组件规范

### 5.1 按钮 (BrandButton)

**Primary 变体**:
```tsx
<BrandButton variant="primary">主要操作</BrandButton>
// 背景: bg-primary-600
// 文字: text-white
// 悬停: hover:bg-primary-700
```

**Secondary 变体**:
```tsx
<BrandButton variant="secondary">次要操作</BrandButton>
// 背景: bg-secondary-100
// 文字: text-secondary-700
// 悬停: hover:bg-secondary-200
```

**Ghost 变体**:
```tsx
<BrandButton variant="ghost">幽灵按钮</BrandButton>
// 背景: transparent
// 文字: text-neutral-600
// 悬停: hover:bg-neutral-100
```

**图标按钮**:
```tsx
<BrandButton iconButton>
// 内边距: p-2
// 圆角: rounded-lg
```

### 5.2 卡片

```tsx
<div className="border border-subtle rounded-lg bg-white p-4">
  {/* 内容 */}
</div>
```

### 5.3 输入框

```tsx
<input
  className="border border-neutral-200 rounded-lg px-4 py-2
         focus:border-primary-300 focus:ring-primary-300
         focus:outline-none focus:ring-2"
/>
```

### 5.4 对话框

```tsx
<div className="fixed inset-0 z-50 flex items-center justify-center">
  <div className="bg-white rounded-lg shadow-xl max-w-md w-full p-4">
    {/* 内容 */}
  </div>
</div>
```

## 7. 动画规范

### 7.1 过渡时长

| 动画类型 | 时长 | CSS 类 |
|---------|------|--------|
| 快速 | 150ms | `transition-[150ms]` |
| 正常 | 200ms | `transition-[200ms]` |
| 慢速 | 300ms | `transition-[300ms]` |

### 7.2 缓动函数

| 用途 | 缓动 |
|------|------|
| 颜色变化 | `ease-in-out` |
| 位置移动 | `ease-out` |
| 透明度变化 | `ease-in` |

### 7.3 常用动画

| 动画 | 说明 | 类 |
|------|------|-----|
| 淡入 | 元素出现 | `animate-fade-in` |
| 淡出 | 元素消失 | `animate-fade-out` |
| 滑入 | 从侧边滑入 | `animate-slide-in` |
| 脉冲 | 强调效果 | `animate-pulse` |

## 9. 空状态设计

### 9.1 原则

1. **视觉引导** - 使用插图或图标传达状态
2. **文字说明** - 清晰解释当前状态
3. **行动引导** - 提供下一步操作
4. **避免空白** - 不留大面积空白

### 9.2 模板

```tsx
<div className="flex flex-col items-center justify-center p-8">
  <Icon className="h-8 w-8 text-neutral-400 mb-2" />
  <p className="text-sm text-neutral-500 text-center">
    暂无内容
  </p>
  <button className="mt-4 text-primary-600 text-sm">
    开始操作
  </button>
</div>
```

## 11. 错误状态设计

### 11.1 Toast 提示

| 类型 | 用途 | 颜色 |
|------|------|------|
| Success | 操作成功 | 绿色 |
| Error | 操作失败 | 红色 |
| Warning | 警告信息 | 橙色 |
| Info | 一般信息 | 蓝色 |

### 11.2 错误页面

当发生严重错误时：
1. 显示友好的错误提示
2. 提供刷新按钮
3. 显示错误代码（供调试）
4. 提供联系支持选项

## 12. 无障碍规范

### 12.1 键盘导航

- 所有交互元素可通过 Tab 访问
- 有清晰的焦点状态
- 快捷键组合不与系统冲突

### 12.2 屏幕阅读器

- 使用语义化 HTML
- 添加 aria-label 到图标按钮
- 提供错误消息的 aria-live 区域

### 12.3 颜色对比

| 文本类型 | 最小对比度 |
|---------|----------|
| 正文 | 4.5:1 |
| 大文本 (18px+) | 3:1 |
| 组件边缘 | 3:1 |
