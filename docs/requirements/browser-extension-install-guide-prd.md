# PRD：浏览器插件手动安装引导功能

> **状态**: 草案  
> **作者**: AI Assistant  
> **日期**: 2026-05-11  
> **目标版本**: v0.x（插件未上架商店期间）

---

## 1. 背景与问题

### 1.1 现状

CreatorWeave 的浏览器插件（Browser Extension）为 Web 端 AI Agent 提供了 **`web_search`** 和 **`web_fetch`** 两项核心能力，使用户能够在对话中搜索互联网、抓取网页内容。

但该插件目前 **未发布到 Chrome Web Store**，只能通过「开发者模式 → 加载已解压的扩展程序」手动安装。

### 1.2 核心问题

| # | 问题 | 影响 |
|---|------|------|
| P1 | **Web 端无任何安装引导 UI** — 用户不知道插件的存在 | 功能发现率 ≈ 0% |
| P2 | **Agent 调用 web_search/web_fetch 失败时仅返回技术错误** | 用户不理解发生了什么，也无法自助解决 |
| P3 | **手动安装流程对非技术用户门槛极高** | 需要下载 zip、解压、找 chrome://extensions、开启开发者模式 |
| P4 | **插件无预构建分发包** | 用户必须 clone 仓库并自行 build |
| P5 | **插件安装后无验证闭环** | 用户不确定安装是否成功 |

### 1.3 目标用户

| 用户类型 | 技术水平 | 占比估算 |
|----------|----------|----------|
| **A. 技术用户** | 会用命令行、了解浏览器扩展 | ~30% |
| **B. 半技术用户** | 会下载文件、会点设置，但不会 CLI | ~50% |
| **C. 非技术用户** | 只会基本浏览器操作 | ~20% |

**本 PRD 重点服务 B 类用户**（覆盖面最广），同时兼顾 A 类和 C 类。

### 3.2 模块一：安装引导入口（多触点）

在多个场景中暴露插件的发现入口，确保用户不会"找不到"。

#### 3.2.1 触点列表

| # | 触点位置 | 触发时机 | 展示形式 | 优先级 |
|---|----------|----------|----------|--------|
| T1 | **顶部通知条（Banner）** | 首次进入 Web 应用，检测到插件未安装 | 固定在页面顶部的浅蓝色横幅：「🌐 解锁网页搜索能力 — 安装浏览器插件」 | P0 |

### 3.4 模块三：插件状态检测 + 实时反馈

#### 3.4.1 检测机制

```typescript
// 状态枚举
type ExtensionStatus = 
  | 'checking'      // 检测中
  | 'installed'     // 已安装并就绪
  | 'not_installed' // 未安装
  | 'error'         // 检测出错

// 检测逻辑（复用现有 web-bridge.tool.ts 的 isWebBridgeAvailable）
function checkExtensionStatus(): ExtensionStatus {
  if (window.__agentWeb?.ready) return 'installed';
  return 'not_installed';
}
```

#### 3.4.2 状态展示位置

| 位置 | 已安装 | 未安装 |
|------|--------|--------|
| 顶部 Banner | 不显示 | 显示安装引导 |
| 设置页插件板块 | ✅ 插件已就绪 (版本 x.x.x) | ⚠️ 未安装 [安装] |
| 对话输入框旁图标 | 🟢 (hover: 插件已连接) | 无图标 |
| Agent 错误卡片 | 不显示 | 替代原有的纯文本错误 |

#### 3.4.3 实时监听

- 插件安装/卸载可能发生在当前页面会话中
- 通过 MutationObserver 或定时轮询（每 5 秒）检测 `window.__agentWeb` 变化
- 状态变化时更新所有 UI 组件
- 安装成功时触发 **祝贺动画**（confetti 🎉 或简洁的 success toast）

### 3.5 模块四：预构建插件包分发

> 这是基础设施前提 — 没有可下载的包，引导流程无从谈起。

#### 3.5.1 构建与托管方案

| 方案 | 描述 | 推荐 |
|------|------|------|
| **A. GitHub Release 附件** | 每次 release 附带 zip，Web 端链接到 GitHub Release 下载 | ⭐ 推荐 — 零成本，版本管理清晰 |
| B. 构建时嵌入 Web 静态资源 | 将 zip 放到 web/public/ 下，随 Web 应用一起部署 | 备选 — 简单但耦合度高 |
| C. CDN 托管 | 上传到 CDN，Web 端从 CDN 下载 | 备选 — 需要额外基础设施 |

#### 3.5.2 推荐方案 A 实现细节

```
1. CI/CD 流程（.github/workflows/build-extension.yml）
   - trigger: push to main / manual dispatch
   - 步骤: npm install → npm run build → npm run zip
   - 产物: chrome-extension.zip 上传到 GitHub Release
   
2. Web 端下载链接
   - https://github.com/{org}/{repo}/releases/latest/download/chrome-extension.zip
   - 自动指向最新版本的 zip

3. 版本号管理
   - 在 wxt.config.ts 中维护版本号
   - 下载 API 可返回当前推荐版本，便于未来提示升级
```

#### 3.5.3 插件包内容规范

```
chrome-extension.zip
├── manifest.json
├── background.js
├── content.js
├── injected.content.js
├── icons/

└── README.txt          ← 新增！面向用户的简要说明
```

**README.txt 内容建议：**

### 3.7 模块六：错误降级处理

| 场景 | 处理方式 |
|------|----------|

### 4.2 状态管理

```typescript
// extensionStore.ts
interface ExtensionState {
  status: 'checking' | 'installed' | 'not_installed' | 'error';
  showBanner: boolean;
  bannerDismissedAt: number | null;  // timestamp
  installGuideStep: number;          // 当前引导步骤 1-5
  installGuideOpen: boolean;
  lastCheckAt: number | null;
}

// Actions
- checkStatus()           // 检测插件状态
- dismissBanner()         // 关闭 banner（7天有效期）
- openInstallGuide()      // 打开安装引导
- closeInstallGuide()     // 关闭安装引导
- goToStep(step: number)  // 切换引导步骤
- resetInstallGuide()     // 重置引导进度
```

### 4.3 检测时机

```
App 初始化
  └── checkStatus()

定时器（每 5s）
  └── checkStatus()

Agent 工具调用前
  └── web-bridge.tool.ts 内部的 isWebBridgeAvailable()

用户点击「重新检测」
  └── checkStatus()

页面 visibilitychange（用户从其他标签页切回）
  └── checkStatus()
```

| `App.tsx` 或顶层 Layout | 挂载 `ExtensionBanner`、状态检测初始化 |
| 设置页 | 新增「浏览器插件」板块 |
| Agent 消息渲染器 | 支持 ExtensionErrorCard 组件类型 |

## 7. 非功能性要求

| 维度 | 要求 |
|------|------|
| **性能** | 插件状态检测不阻塞主线程；轮询间隔 ≥ 2s |
| **兼容性** | Chrome 96+、Edge 96+；Firefox 作为可选支持 |
| **国际化** | 引导流程支持中/英双语 |
| **无障碍** | 引导弹窗支持键盘导航；步骤组件有正确的 ARIA 标签 |
| **安全** | 下载链接使用 HTTPS；插件包有 SHA256 校验（可选） |
| **响应式** | 引导弹窗在移动端显示"请在电脑端安装"提示 |
| **持久化** | 引导进度、Banner 关闭状态存 localStorage，清除缓存后可恢复 |
