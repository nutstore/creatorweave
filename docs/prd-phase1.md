## 1. 产品概述

### 1.1 产品简介

Browser File System Analyzer 是一款**完全在浏览器沙盒中运行**的本地文件系统分析工具。用户无需安装任何软件，通过浏览器即可分析本地文件夹的文件大小和结构信息。

### 1.2 核心价值主张

| 价值点 | 说明 |
|--------|------|
| **零安装** | 打开浏览器即可使用，无需下载安装 |
| **隐私安全** | 所有计算在本地完成，数据不上传服务器 |
| **高性能** | 使用 Rust + WebAssembly 处理计算密集型任务 |
| **现代化** | 响应式 UI，支持深色模式，流畅的交互体验 |

### 1.3 Phase 1 目标

**核心目标**：实现文件夹大小分析功能

1. 用户可以通过浏览器原生对话框选择本地文件夹
2. 应用递归遍历所有子文件夹
3. 收集每个文件的大小信息（仅读取元数据，不读取文件内容）
4. 使用 WASM 模块进行累加计算
5. 实时显示分析进度和结果

### 1.4 成功指标

| 指标 | 目标值 |
|------|--------|
| 功能完整性 | 核心功能 100% 实现 |
| 浏览器兼容性 | Chrome 86+, Edge 86+, Opera 72+ |
| UI 响应性 | 遍历过程中 UI 保持可交互 |
| 错误处理 | 所有可预见的错误都有友好的用户提示 |
| 测试覆盖率 | 核心逻辑 >= 80% |

## 2. 功能需求

### 2.1 功能列表

#### FR-1: 选择文件夹

**用户故事**：作为用户，我想要选择一个本地文件夹，以便分析其中的文件。

**功能描述**：

| 属性 | 值 |
|------|-----|
| 优先级 | P0 (必须有) |
| 触发方式 | 点击"选择文件夹"按钮 |
| 依赖 | File System Access API (`showDirectoryPicker`) |

**交互流程**：

```
用户点击"选择文件夹"按钮
         ↓
    调用浏览器 API
    window.showDirectoryPicker()
         ↓
    用户在系统对话框中选择文件夹
         ↓
    用户授予读取权限
         ↓
    返回 FileSystemDirectoryHandle
         ↓
    开始分析流程
```

**异常处理**：

| 异常场景 | 处理方式 |
|----------|----------|
| 用户取消选择 | 显示提示"已取消选择文件夹"，返回初始状态 |
| 浏览器不支持 API | 显示提示"您的浏览器不支持文件系统访问 API，请使用 Chrome、Edge 或 Opera 最新版本" |
| 权限被拒绝 | 显示提示"需要读取权限才能分析文件夹" |

#### FR-2: 递归遍历文件夹

**用户故事**：作为系统，我需要遍历用户选择的所有子文件夹，以便收集完整的文件信息。

**功能描述**：

| 属性 | 值 |
|------|-----|
| 优先级 | P0 (必须有) |
| 执行方式 | 异步递归遍历 |
| 数据收集 | 文件大小 (file.size) |

**遍历逻辑**：

```typescript
async function* traverseDirectory(dirHandle: FileSystemDirectoryHandle) {
  for await (const entry of dirHandle.values()) {
    if (entry.kind === 'file') {
      const file = await entry.getFile();
      yield {
        name: file.name,
        size: file.size,
        type: file.type,
        lastModified: file.lastModified
      };
    } else if (entry.kind === 'directory') {
      yield* traverseDirectory(entry);  // 递归
    }
  }
}
```

**性能要求**：

- 遍历过程不阻塞主线程 UI
- 支持深度嵌套的文件夹结构
- 支持包含数万个文件的文件夹

**异常处理**：

| 异常场景 | 处理方式 |
|----------|----------|
| 文件被占用/无法访问 | 跳过该文件，记录警告，继续处理其他文件 |
| 文件夹过深 | 限制最大递归深度（如 100 层），超过则跳过并警告 |
| 循环链接（符号链接） | 检测并跳过，防止无限循环 |

#### FR-3: WASM 累加计算

**用户故事**：作为系统，我需要使用 WASM 模块进行累加计算，以便展示高性能计算能力。

**功能描述**：

| 属性 | 值 |
|------|-----|
| 优先级 | P0 (必须有) |
| 实现方式 | Rust 编译为 WASM |
| 接口方式 | wasm-bindgen 导出的 JavaScript API |

**WASM 模块接口**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `constructor()` | - | `FileAnalyzer` | 创建分析器实例 |
| `add_files(sizes)` | `number[]` | `void` | 批量添加文件大小 |
| `get_total()` | - | `number` | 获取总字节数 |
| `get_count()` | - | `number` | 获取文件数量 |
| `get_average()` | - | `number` | 获取平均文件大小 |
| `reset()` | - | `void` | 重置状态 |

**数据流**：

```
JavaScript 遍历文件夹
        ↓
收集文件大小数组
[file1.size, file2.size, ...]
        ↓
分批传递给 WASM
analyzer.add_files(batch)
        ↓
WASM 执行累加
        ↓
查询结果
const total = analyzer.get_total()
```

**批量策略**：

- 每收集 100 个文件大小传递一次（可配置）
- 避免频繁跨边界调用带来的性能开销
- 支持分批更新进度显示

## 5. 边缘情况处理

### 5.1 空文件夹

| 场景 | 处理方式 |
|------|----------|
| 用户选择了空文件夹 | 显示"此文件夹为空"的提示信息 |
| 文件夹只有子文件夹，没有文件 | 显示"未找到文件，包含 X 个子文件夹" |

### 5.2 大文件夹

| 场景 | 处理方式 |
|------|----------|
| 文件数量超过 10,000 | 正常处理，显示进度 |
| 文件夹总大小超过 10GB | 正常处理，使用 GB 单位显示 |
| 遍历时间超过 30 秒 | 显示"正在处理大型文件夹，请稍候..."提示 |

### 5.3 深层嵌套

| 场景 | 处理方式 |
|------|----------|
| 嵌套深度超过 100 层 | 停止递归，显示警告"已达到最大深度限制" |
| 检测到循环链接（符号链接） | 跳过该链接，记录警告 |

### 5.4 权限问题

| 场景 | 处理方式 |
|------|----------|
| 子文件夹无访问权限 | 跳过该文件夹，显示警告 |
| 文件被其他程序占用 | 跳过该文件，记录到警告列表 |

### 5.5 网络问题

虽然应用完全在本地运行，但 WASM 模块加载需要网络：

| 场景 | 处理方式 |
|------|----------|
| WASM 模块加载失败 | 显示"资源加载失败，请检查网络连接" |
| 加载超时 (> 10 秒) | 显示"加载超时，请刷新页面重试" |
