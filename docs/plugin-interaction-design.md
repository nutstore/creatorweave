## 1. 概述

### 1.1 文档目的

本文档定义了 Browser File System Analyzer 插件系统的用户体验设计，包括界面布局、交互流程、视觉规范和无障碍要求，确保用户能够轻松管理和使用插件来扩展文件分析功能。

### 1.2 目标用户

| 用户类型 | 描述 | 核心需求 |
|----------|------|----------|
| **普通用户** | 需要分析本地文件的非技术用户 | 简单易用，一键安装插件，清晰的结果展示 |
| **技术用户** | 开发者、分析师，需要自定义分析 | 能够上传自定义插件，查看详细执行信息 |
| **插件开发者** | 创建和分发插件的人员 | 清晰的开发文档，调试工具，插件测试界面 |

### 1.3 设计原则

| 原则 | 描述 |
|------|------|
| **渐进式披露** | 基础功能简单可见，高级功能按需展示 |
| **即时反馈** | 每个操作都有明确的视觉反馈 |
| **安全透明** | 清晰展示插件权限和安全状态 |
| **容错性** | 错误操作可恢复，失败有明确指引 |
| **一致性** | 与现有界面保持统一的交互模式 |

## 2. 信息架构

### 2.1 页面结构

```
Browser File System Analyzer
│
├── 首页 (Folder Selection)
│   ├── 选择文件夹按钮
│   └── 已有项目快速入口
│
├── 分析配置页 (Analysis Config)
│   ├── 文件夹信息卡片
│   ├── 插件选择区
│   │   ├── 内置分析 (始终启用)
│   │   ├── 已安装插件列表
│   │   └── 添加新插件入口
│   └── 开始分析按钮
│
├── 分析进行中 (Analysis Progress)
│   ├── 总体进度条
│   ├── 当前文件信息
│   ├── 插件执行状态
│   └── 取消/暂停按钮
│
├── 结果展示页 (Results)
│   ├── 总览仪表板
│   ├── 内置分析结果
│   ├── 插件结果标签页
│   └── 导出/重新分析
│
└── 插件管理页 (Plugin Management)
    ├── 插件商店 (未来)
    ├── 已安装插件
    ├── 上传新插件
    └── 插件详情设置
```

### 2.2 导航设计

**主导航** - 顶部固定导航栏

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo] Browser FS Analyzer    [插件]  [关于]  [设置]      │
└─────────────────────────────────────────────────────────────┘
```

**面包屑导航** - 显示当前流程位置

```
选择文件夹 → 配置分析 → 分析中 → 查看结果
```

**返回/前进** - 每个页面底部有明确的上一步/下一步操作

### 2.3 信息层级

| 层级 | 内容 | 视觉处理 |
|------|------|----------|
| L0 - 关键操作 | 主要CTA按钮（开始分析、上传插件） | 大尺寸、主色调、突出显示 |
| L1 - 核心信息 | 文件数量、分析结果摘要 | 大字号、加粗 |
| L2 - 次要信息 | 插件描述、文件路径详情 | 常规字号 |
| L3 - 辅助信息 | 帮助文本、提示信息 | 小字号、浅色 |

useEffect(() => {

    firstFocusableRef.current?.focus();

    // 捕获 Tab 键
    const handleTab = (e: KeyboardEvent) => {

        trapFocus(e, focusableElements);
