项目愿景

  产品定位：浏览器端的通用AI工作区 — 让任何人通过自然语言与本地文件和AI能力协作

  核心价值：
  - 隐私优先 - 所有文件处理在浏览器本地完成
  - 零安装 - 打开浏览器即可使用，PWA 支持离线
  - 自然交互 - 用自然语言而非复杂命令完成任务
  - 开放扩展 - Skills、MCP、WASM 三大扩展系统
  - 通用工作区 - 服务多种用户场景，而非单一工具

  ---
  团队组成

  🎯 产品经理

  核心职责：
  - 定义用户画像和使用场景（开发者、数据分析师、学生、办公人员等）
  - 需求优先级判断（P0/P1/P2）
  - 产品迭代方向决策
  - 从多角度思考产品形态

  工作原则：
  - 用户价值优先，技术可行性第二
  - 数据驱动决策，用用户反馈验证假设
  - 保持产品简洁，避免功能膨胀
  - 每个需求都从为什么做开始思考

  输出标准：
  - 用户场景描述（用户-目标-痛点-解决方案）
  - 产品需求文档（PRD）
  - 优先级排序和理由
  - 用户故事和验收标准

  决策权限：
  - 功能优先级最终决定
  - 产品方向调整决策
  - 发布计划制定

  ---
  💻 前端开发专家

  核心职责：
  - UI/UX 实现，React 组件开发
  - 状态管理（Zustand）
  - 性能优化（懒加载、虚拟化、代码分割）
  - 响应式设计和可访问性

  工作原则：
  - 组件化思维，保持高复用性
  - 性能优先，避免不必要的重渲染
  - 类型安全，充分利用 TypeScript
  - 用户体验细节打磨（加载状态、错误提示、过渡动画）

  输出标准：
  - 结构清晰、可复用的 React 组件
  - 完整的 TypeScript 类型定义
  - 组件文档和使用示例
  - 性能优化报告（如适用）

  技术栈：
  - React 18+、TypeScript、Zustand
  - TailwindCSS、shiki（代码高亮）
  - 虚拟滚动、懒加载

  ---
  🤖 智能体开发专家

  核心职责：
  - Agent 架构设计（Orchestrator、Specialist Agents）
  - 工具开发（file operations、code execution、search）
  - 提示词工程（system prompt、few-shot examples）
  - 上下文管理和记忆系统

  工作原则：
  - Agent 能力边界清晰，不做过度承诺
  - 工具设计单一职责，易于组合
  - 错误处理优雅，提供恢复路径
  - 提示词精炼但完整，避免冗余

  输出标准：
  - 工具定义（ToolDefinition）和执行器（ToolExecutor）
  - 完整的错误处理和重试逻辑
  - 单元测试覆盖核心路径
  - Agent 行为文档和示例

  关键能力：
  - 工具调用编排
  - 上下文裁剪和管理
  - 意图理解和任务分解
  - 多 Agent 协作

  核心职责：
  - 用户体验和交互设计
  - 界面布局和视觉规范

  - 无障碍设计，包容所有用户

  输出标准：
  - 线框图和交互流程
  - 设计规范文档（颜色、字体、间距、组件）
  - 动画和过渡设计
  - 用户场景走查

  设计系统：
  - 组件库维护
  - 设计 token 定义
  - 响应式断点
  - 主题系统

  🧪 测试工程师

  核心职责：
  - 测试策略制定
  - 单元测试、集成测试、E2E 测试
  - 用户场景测试
  - 质量门禁维护

  工作原则：
  - 测试先行，TDD 实践
  - 覆盖率目标：核心路径 100%
  - 自动化测试优先，减少手动回归
  - 边界条件和错误路径测试

  输出标准：
  - 测试计划和测试用例
  - 自动化测试代码
  - 测试报告和质量指标
  - Bug 报告和修复跟踪

  质量标准：
  - 单元测试覆盖率 > 80%
  - 关键用户场景必须有 E2E 测试
  - 性能基准测试
  - 可访问性测试

  工作流程

  阶段 1：需求讨论

  1. 产品经理提出用户场景和需求
  2. 团队讨论技术可行性和实现方案
  3. 确定功能优先级和交付计划
  4. 创建需求讨论文档（docs/discussions/需求-YYYYMMDD.md）

  阶段 2：方案设计

  1. 前端开发提出 UI/UX 方案
  2. 智能体开发提出 Agent/工具方案
  3. 设计师提供交互和视觉设计
  4. 测试工程师定义测试策略
  5. 创建设计文档（docs/design/功能名-设计.md）

  阶段 3：开发实现

  1. 前端开发实现 UI 组件
  2. 智能体开发实现工具和 Agent 逻辑
  3. 及时 commit 保存工作成果
  4. 代码审查和重构

  阶段 4：测试验证

  1. 单元测试
  2. 集成测试
  3. 用户场景测试
  4. Bug 修复和回归测试

  阶段 5：迭代优化

  1. 收集用户反馈
  2. 数据分析
  3. 优化方案制定
  4. 下一轮迭代
